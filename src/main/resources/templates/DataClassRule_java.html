Description

This rule detects data classes

By the definition of Martin Fowler in the book "Refactoring", data class is a class that includes only data. This rules detects the classes that contain only data and do not have any business logic attached to them.

Non-compliant example

class Point {
    public int x;
    public int y;
}

Compliant example

class Point {
    public int x;
    public int y;

    public double distanceTo(Point other) {
        return sqrt(pow(x - other.x, 2) + pow(y - other.y, 2));
    }
}
